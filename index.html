HTML

  • HTML = Hypertext Mark Up Language
  • Lays out the base aspects of a webpage using elements
  • Detail certain areas with a header, body, and footer with (head), (body), and (footer) elements
  • Create subheadings with (h1), (h2), ... elements
  • Can create lists using (ul) and (li) elements
  • Can add pictures and link to other scipts like CSS or JavaScript

CSS

  • CSS = Cascading Style Sheets
  • CSS is used to alter the appearance of HTML files
  • Can change many aspects of the webpage including pictures, headers, footers, size, color, etc.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • Git is where code can be stored and linked for work on multiple machines
  • cd: chnages the file we are working in
  • touch: creates a new file in the directory
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull: updates local directory to match Git
  • git push: uploads local changes to stored Git code
  • git add: adds changed files to staging area in Git

JavaScript

  • JavaScript makes a webpage more interactive
  • Viewed in browser but inspecting DevTools on the page
  • A variable is a named container that allows us to store data in our code.
  • An array is a variable that holds a group of data
  • Control flow is the order in which a computer executes code in a script.
  • for loops can be integrated to shorten code and more easily use arrays
  • Can add comments using /* and */ around text or // before a single line of text
